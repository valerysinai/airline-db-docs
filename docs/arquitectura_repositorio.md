# Arquitectura del Repositorio

## Estructura

El proyecto se divide en dos repositorios:

- airline-db-project (código)
- airline-db-docs (documentación)

## Estrategia de ramas

- main: producción
- qa: pruebas
- develop: desarrollo

## Flujo de trabajo

develop → qa → main

## Ambientes

- dev: entorno local con Docker
- qa: pruebas de integración
- prod: entorno estable
